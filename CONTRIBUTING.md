# Contributing

Thanks for helping build AI Engineer Arena.

## Contribution principles

- Keep changes focused and easy to review.
- Prefer educational clarity over cleverness.
- Include tests and documentation updates with behavior changes.
- Follow existing naming and folder conventions.

## Problem author checklist

Each new problem should include:

- `problem.yaml` metadata
- `statement.md` with constraints and examples
- `tests/public.json` simple logic tests
- `tests/hidden.json` stress and edge tests
- `editorial.md` with complexity analysis

## Editorial quality requirements

- include at least one baseline and one optimized approach
- include time and space complexity
- explain trade-offs and interview recommendation

## Commit and PR guidance

- use small, meaningful commits
- keep PR scope limited to one coherent objective
- add validation evidence (commands + outputs summary)
- avoid committing generated artifacts unless required
